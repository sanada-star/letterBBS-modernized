# LetterBBS プロジェクト憲法 (ANTIGRAVITY.md)

## 1. プロジェクト概要
**LetterBBS** は、KENT WEB様で公開されている「Patio」をベースに、2025年スタイルの「オープン私書箱」として近代化・高機能化したWebアプリケーションです。
PBC（Play-by-Chat）やPBWでの「キャラクター交流」や「秘匿会話」に特化しています。

## 2. コアバリュー & デザイン原則 (Vibe Coding)
- **Vibe Coding**: 厳密なエンジニアリングよりも、ユーザー体験と「手触り」を最優先します。「お店のセキュリティ」「料理」などの比喩を用いて分かりやすく説明します。
- **MVP原則**: 常に最小限の実用構成から始め、段階的に機能を拡張します。
- **Safety**: 破壊的な操作は必ず確認を行い、既存データを守ります。

## 3. 技術スタック
- **Frontend**: HTML5, Vanilla CSS (Flexbox/Grid), Vanilla JavaScript (LocalStorage活用)
- **Backend**: Perl (Legacy CGI) - レンタルサーバーでの動作互換性を維持。
- **Design**: "Pop/Chaotic" (Mochiy Pop One / Zen Maru Gothic) - ビビッドでモダンな世界観。

## 4. 主な機能 (v3.0 Current / v3.1 Dev)
- **タイムライン表示**: 1対1のチャット形式で見やすく表示。
- **文通デスク**: 複数人への返信を下書き・一括送信する機能。
- **通知タスク機能**: 自分宛ての手紙を常駐通知し、ToDoリストとして管理（v3.0）。
- **テーマ切り替え**: ユーザーがCSSテーマ（Pop/Gloomy/Simple）を選択可能。
- **メモリーボックス**: 過去ログのHTMLダウンロードとローカル閲覧。
- **[Next] v3.1開発機能**: より高度なインタラクションや管理機能の拡充を予定（テスト領域）。

### 開発ガイドライン
1.  **レガシーバックエンドの尊重**: CGIロジックはPerlです。無理に他言語へ書き換えず、既存の資産を活かしてください。
2.  **フロントエンドの近代化**: CSS/JSは最新の標準技術を積極的に採用してください。レトロなバックエンド上で、モダンなUXを実現するのがこのプロジェクトの醍醐味です。
3.  **ユーザーコミュニケーション**: ユーザーは「日本語」での対話を求めています。親しみやすく、かつプロフェッショナルな態度で接してください。
4.  **デプロイ時の注意**: CGIファイルのパーミッション設定（705/755など）は非常に重要です。変更後は必ず確認を促してください。
